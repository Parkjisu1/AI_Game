## 3. 게임플레이 시스템 (Gameplay)

### 보드 구성 (1080x1920 기준)

| 영역 | Y 범위 | 설명 |
|------|--------|------|
| 상단 바 | 0~150 | 코인 수량(좌상), Level N 표시(중앙상), 설정 기어(우상) |
| 게임 보드 | 250~1300 | 3D 주차장, 색상별 자동차 배치 |
| 홀더 | 1350~1450 | 7칸 슬롯 (탭한 자동차가 이동하는 곳) |
| 부스터 바 | 1800~1870 | 4개 부스터 버튼 |

### 자동차 시스템

| 항목 | 확정 값 | 소스 |
|------|---------|------|
| 자동차 색상 | **8종 이상**: 빨강, 주황, 노랑, 초록, 시안, 파랑, 보라, 핑크 | 스크린샷 관찰 |
| 자동차 크기 | 약 100~130px 너비, 80~120px 높이 | 스크린샷 실측 |
| 매칭 규칙 | **같은 색 3대** → 소멸 | 게임플레이 관찰 |
| 홀더 슬롯 | **7칸** | 스크린샷 실측 |
| 홀더 가득 조건 | 7칸 모두 차있고 매칭 불가 = **게임 오버** | 실패 화면 캡처 |

### 자동차 상태 (활성/비활성 판정)

| 상태 | 판정 기준 | 탭 가능 여부 |
|------|----------|------------|
| **ACTIVE (활성)** | 자동차의 **눈/얼굴/앞유리가 보임** | O 탭 가능 |
| **BLOCKED (비활성)** | 앞 자동차에 가려져 **눈이 안 보임** | X 탭 불가 |
| **STACKED (x2~x5)** | 숫자 표시 = 여러 대 쌓임, 맨 위만 탭 가능 | 상위만 탭 가능 |
| **MYSTERY (?)** | 물음표 표시 = 색상 미확인, 탭하면 공개 | O (홀더 3칸+ 비어있을 때만 권장) |
| **LOCKED** | 자물쇠 아이콘 = 특정 매칭 후 해금 | X 해금 전까지 불가 |

> **CRITICAL**: 활성/비활성 판정 핵심은 **"눈(eyes)이 보이는가"**. 자동차 캐릭터에 눈이 달려있으며, 눈이 보이면 활성, 앞차에 가려져 눈이 안보이면 비활성.

### 부스터 시스템 (하단 바 y~1830)

| 위치 (x) | 부스터 | 비용 | 기능 |
|----------|--------|------|------|
| 108 | **Undo** (되돌리기) | 무료 (2회 제한) | 마지막 자동차를 홀더에서 보드로 되돌림 |
| 324 | **Magnet** (자석) | 무료 (1회 제한) | 보드에서 같은 색 3대를 자동 매칭 |
| 540 | **Shuffle** (셔플) | 900 코인 | 보드 자동차 재배치 |
| 756 | **Rotate** (회전) | 600 코인 | 보드 자동차 회전 |

> **위험**: Shuffle(900코인), Rotate(600코인)은 유료 — 자동 플레이 시 절대 탭 금지

### 게임플레이 전략 (관찰 기반)

```
1. 홀더 확인 → 같은 색 2대 있으면 보드에서 3번째를 찾아 즉시 탭
2. 매칭 불가 시 → 앞줄(ACTIVE) 자동차부터 탭하여 뒷줄 해금
3. 홀더 5칸 이상 → Undo 부스터 사용 권장
4. MYSTERY(?) 자동차 → 홀더 3칸+ 비어있을 때만 탭
5. STACKED(x2~x5) → 맨 위 자동차만 탭 가능, 아래 색상 미확인
```

|------|------|------|----------|
| Vision 방식 | Read tool (PNG 바이너리) | Anthropic API (base64 이미지) | 좌표 정확도 대폭 향상 |
| 화면 분류 | Vision AI (~15초) | 로컬 pHash + 픽셀 체크 (~0.1초) | 사이클 속도 150x 향상 |
| 팝업 처리 | 단일 X 좌표 | 다중 X 좌표 + back 버튼 + force-stop | 무한 루프 방지 |
| 광고 복귀 | 없음 (다른 게임에 갇힘) | 패키지 감지 + force-stop + relaunch | 광고 게임 탈출 |
| 자동차 인식 | Vision AI 전체 의존 | 색상 히스토그램 + 눈 감지 | 로컬 판정으로 속도/정확도 향상 |

### 교훈 (Lessons Learned)

| 번호 | 교훈 | 대응 방안 |
|------|------|----------|
| 1 | Read tool로 PNG 읽기는 실제 비전이 아님 | Anthropic Messages API + base64 이미지 사용 |
| 2 | 광고가 다른 게임으로 전환할 수 있음 | 패키지명 감지(dumpsys) + force-stop + relaunch |
| 3 | Vision AI가 "이건 다른 게임"이라고 해도 JSON 반환함 | parse 단계에서 "not CarMatch" 키워드 감지 시 게임 복귀 로직 |
| 4 | 단일 X 좌표로 팝업 닫기는 실패할 수 있음 | 다중 좌표 시도 + back 버튼 + 5회 실패 시 force-stop |
| 5 | detect_screen에 Vision AI 사용하면 너무 느림 | 로컬 pHash + 픽셀 색상 체크로 <0.1초 분류 |
| 6 | 사이클당 2분은 실시간 게임에 너무 느림 | 로컬 분류 + 배치 탭으로 사이클 5초 이하 목표 |
