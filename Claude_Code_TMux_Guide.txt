========================================
  Claude Code TMux 사용 가이드
========================================

[ 1. 설치 ]

# Claude Code 설치 (WSL2 환경)
curl -fsSL https://claude.ai/install.sh | bash

# tmux 설치 (Ubuntu/Debian)
sudo apt-get install tmux

# 설치 확인
claude doctor
which tmux


[ 2. 인증 ]

# 프로젝트 폴더로 이동 후 실행
cd /mnt/e/AI
claude

# 처음 실행 시 Claude.ai 계정으로 로그인


[ 3. 설정 ]

설정 파일 위치: ~/.claude/settings.json

필요한 설정:
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  },
  "teammateMode": "tmux"
}

* teammateMode를 "tmux"로 설정하면 Agent들이 tmux split pane으로 실행됨
* "in-process"로 설정하면 단일 터미널에서 실행


[ 4. 기본 실행 ]

# 대화형 모드 시작
cd /mnt/e/AI
claude

# 이전 세션 이어서 계속
claude --continue

# 특정 세션 재개
claude --resume <session-id>

# 헤드리스 모드 (비대화형, 자동 실행)
claude -p "명령어" --allowedTools "Read,Edit,Bash" --output-format json

# 버전 업데이트
claude update


[ 5. Agent Team 실행 방법 ]

Claude Code 프롬프트에서 아래처럼 요청:

  "게임을 만들어줘. [게임 설명]. Agent Team으로 병렬 진행해줘. Coder N명."

TMux 모드에서는 각 Agent가 별도 tmux pane으로 실행되어
Designer, Main Coder, Sub Coder, Validator의 작업을 실시간으로 볼 수 있음.

Agent 팀 구성 (CLAUDE.md 기준):
  - Lead (Opus)      : PM - 태스크 분배/조율
  - Designer (Sonnet) : 기획서 생성
  - Main Coder (Opus) : Core 아키텍처, 핵심 시스템
  - Sub Coder (Sonnet) : Main Coder 패턴 따라 노드 구현
  - Validator (Sonnet) : 코드 검증, 피드백 생성


[ 6. 프로젝트 구조 ]

E:\AI\
  CLAUDE.md              - 프로젝트 지침서 (자동 로드됨)
  .claude\
    agents\              - Custom Agent 정의 파일들
      designer.md
      main-coder.md
      sub-coder.md
      validator.md
      db-builder.md
    settings.json        - 설정 파일
  db\                    - Base/Expert Code DB
  projects\              - 프로젝트별 output
  History\               - 개발 히스토리
  Feedback\              - 워크플로우 피드백


[ 7. 슬래시 명령어 ]

Claude Code 내에서 사용:
  /help              - 도움말
  /clear             - 컨텍스트 초기화
  /config            - 설정 확인/변경
  /compact            - 컨텍스트 압축
  /cost              - 현재 세션 비용 확인

커스텀 스킬 (CLAUDE.md에 정의된 명령):
  /parse-source [path]     - 소스코드 파싱 후 DB 저장
  /generate-design [input] - 기획서/명세서 생성
  /generate-code [yaml]    - 코드 생성
  /validate-code [path]    - 코드 검증


[ 8. TMux 세션 관리 ]

# 활성 세션 목록 확인
tmux ls

# 특정 세션 종료
tmux kill-session -t <session-name>

# 모든 세션 종료
tmux kill-server

# TMux 기본 단축키 (Ctrl+B가 prefix)
Ctrl+B, %     : 세로 분할
Ctrl+B, "     : 가로 분할
Ctrl+B, 방향키 : pane 이동
Ctrl+B, d     : 세션에서 분리 (detach)
Ctrl+B, x     : 현재 pane 종료


[ 9. 트러블슈팅 ]

문제: TMux가 작동하지 않음
해결: claude --teammate-mode in-process  (in-process 모드로 전환)

문제: 세션이 끊김
해결: claude --continue  또는  claude --resume <id>

문제: Agent가 응답하지 않음
해결: tmux ls로 확인 후 orphan 세션 정리

문제: 컨텍스트가 너무 커짐
해결: /compact 명령어로 압축
