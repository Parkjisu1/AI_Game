## 핵심 발견

### 1. Unity 3D 입력 시스템 (최대 기여)
- ADB `input tap`은 Unity 3D 오브젝트에 무반응
- `monkey -p [package]` 스크립트만 3D 터치 가능
- 이는 Unity의 EventSystem이 Android의 MotionEvent를 직접 수신하되,
  input tap이 생성하는 이벤트는 Unity 렌더링 레이어까지 도달하지 않기 때문
- **부작용**: monkey -p가 Android Intent를 트리거 → 인터스티셜 광고 발생

### 2. 홀더 메커니즘 정밀 분석
- 7슬롯 x좌표 정밀 측정: [173, 296, 419, 543, 667, 793, 912]
- 색상 기반 상태 추적 (빈칸 = 어두운 회색)
- Late Arrival 패턴 발견: 차량 진입 + 매치가 동시 발생 시 중간 상태 캡처

### 3. 세틀 타임 3단계 체계
- 홀더 상태에 따른 적응형 대기 시간
- 5+ 상태에서 추가 2초 재확인 필수 (오버플로우 방지)
