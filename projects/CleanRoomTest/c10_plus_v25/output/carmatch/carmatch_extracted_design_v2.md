## 1. 게임 개요

| 항목 | 값 | 소스 |
|------|-----|------|
| 게임명 | CarMatch | 관찰 |
| 패키지 | com.grandgames.carmatch | ADB |
| 엔진 | Unity (3D) | 입력 실험 |
| 장르 | Casual Puzzle (3D 주차장 매칭) | 확정 |
| 해상도 | 1080x1920 (portrait) | 캡처 확인 |
| 레벨 | 117 (관찰 시점) | 확정 |
| Virtual Player 장르 | puzzle (14 screen types) | test_orchestrator |

### 게임 컨셉
- 3D 주차장에 배치된 차량을 탭하여 하단 홀더로 이동
- 같은 색 차량 3대가 홀더에 모이면 자동 매치(소멸)
- 모든 차량을 클리어하면 레벨 클리어
- 홀더 7칸이 가득 차면 게임 오버

## 2. 코어 루프

```
[차량 탭] → 홀더로 이동 (애니메이션)
    ↓
[같은 색 3대?] → YES → 자동 매치 소멸
    ↓ NO             ↓
[홀더 가득?]     [보드 비었는가?]
    ↓ YES            ↓ YES
[GAME OVER]      [LEVEL CLEAR]
    ↓ NO             ↓ NO
[계속 탭]         [계속 탭]
```

## 9. 입력 시스템 (핵심 발견)

| 방법 | 3D 차량 | UI 버튼 | Intent |
|------|:------:|:------:|:------:|
| monkey -p | O | O | O |
| input tap | X | O | X |
| sendevent | X | X | — |

**기술적 원인**: Unity EventSystem은 Android View 레이어의 MotionEvent를 직접 수신하지 않음.
monkey -p는 패키지 컨텍스트에서 이벤트를 주입하여 Unity 렌더링 레이어까지 전달.

## 11. 자동화 도구

### carmatch_ai.py (~1,050줄)

| 컴포넌트 | 기능 |
|---------|------|
| detect_screen() | 7종 화면 상태 분류 |
| read_holder_colors() | 홀더 7슬롯 색상 추출 |
| find_red_x() | 광고 X 버튼 동적 탐지 |
| strategy_engine() | 리스크 기반 4단계 전략 |
| monkey_tap() | 3D 오브젝트 탭 |
| input_tap() | UI 버튼 탭 |
| handle_ad() | 광고 닫기 루틴 |
| play_level() | 전체 레벨 플레이 루프 |

### 실행 방법
```bash
python carmatch_ai.py               # 끝까지 (lives 소진 시 25분 대기)
python carmatch_ai.py --hours 8     # 8시간 자율 플레이
python carmatch_ai.py --levels 5    # 5레벨 후 종료
```

| 파일 | 설명 |
