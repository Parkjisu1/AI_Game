## 1. 게임 개요

| 항목 | 값 | 소스 |
|------|-----|------|
| 게임명 | CarMatch | 관찰 |
| 패키지 | com.grandgames.carmatch | ADB |
| 엔진 | Unity (3D) | 입력 실험으로 확인 |
| 장르 | Casual Puzzle (3D 주차장 매칭) | 확정 |
| 해상도 | 1080x1920 (portrait) | 캡처 확인 |
| 레벨 | 117 (관찰 시점) | 확정 |

### 게임 컨셉
- 3D 주차장에 배치된 차량을 탭하여 하단 홀더로 이동
- 같은 색 차량 3대가 홀더에 모이면 자동 매치(소멸)
- 모든 차량을 클리어하면 레벨 클리어
- 홀더 7칸이 가득 차면 게임 오버

---

## 2. 코어 루프

```
[차량 탭] → 홀더로 이동 (애니메이션)
    ↓
[같은 색 3대?] → YES → 자동 매치 소멸
    ↓ NO             ↓
[홀더 가득?]     [보드 비었는가?]
    ↓ YES            ↓ YES
[GAME OVER]      [LEVEL CLEAR]
    ↓ NO             ↓ NO
[계속 탭]         [계속 탭]
```

## 5. 세틀 타임 시스템

차량 탭 후 홀더 진입 + 매치 완료까지 대기 시간:

| 홀더 상태 | 세틀 시간 | 이유 |
|----------|----------|------|
| 0-2/7 칸 | 4.0초 | 기본 차량 이동 |
| 3-4/7 칸 | 5.0초 | 이동 + 매치 가능 |
| 5+/7 칸 | 6.0초 + 2초 재확인 | 오버플로우 치명적 |

### Late Arrival 패턴
- 차량이 홀더에 진입하면서 동시에 매치가 발생
- 첫 스캔: 5/7 → 2초 후 재스캔: 4/7 (매치 완료)
- 이 패턴을 무시하면 과도한 신중 모드 진입 → 효율 저하

### Ad-for-Life 플로우
```
Lives = 0 → Play 탭 → "More Lives" 팝업
    → "+1 Life" (541, 1250) 탭
    → 30초 광고 시청
    → X 버튼으로 닫기
```

## 10. 입력 시스템 (핵심 기술 발견)

### ADB 입력 방법 비교

| 방법 | 3D 차량 | UI 버튼 | Intent |
|------|:------:|:------:|:------:|
| `monkey -p` 스크립트 | O | O | O (광고) |
| `input tap` | X | O | X |
| `sendevent` | X | X | — |
| `monkey` (no -p) | X | X | X |

### 기술적 원인
Unity 게임은 자체 EventSystem으로 터치 이벤트를 처리.
ADB `input tap`이 생성하는 MotionEvent는 Android View 레이어에서 처리되지만,
Unity의 3D 렌더링 레이어까지 도달하지 않음.
`monkey -p`는 패키지 컨텍스트에서 이벤트를 주입하여 Unity까지 전달됨.
