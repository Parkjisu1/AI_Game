## 2. 개선안: v3.0 "Human Capture + AI Analysis"

### 2.1 핵심 변경: ADB 자동화 제거 → 인간 캡처

**Before (v2.5):**
```
[ADB 자동화] → 스크린샷 캡처 → [Claude Vision] → 분석
   ↑ 문제: Unity 게임에서 UI 클릭 실패
```

**After (v3.0):**
```
[인간이 직접 플레이] → 스크린샷 제공 → [Claude Vision] → 분석
   ↑ 해결: 모든 UI 화면 접근 가능
```

| 비교 항목 | v2.5 (ADB) | v3.0 (Human Capture) |
|-----------|-----------|---------------------|
| UI 접근성 | Unity 게임 불가 | 모든 게임 100% |
| 캡처 품질 | 해상도 고정 (540×960) | 디바이스 네이티브 해상도 |
| 세션 시간 | ~10분 (ADB 대기) | ~5분 (인간이 빠름) |
| 법적 리스크 | APK 추출 필요 | 없음 |
| 확장성 | 에뮬레이터 설치 필요 | 스마트폰만 있으면 됨 |
| 비용 | ADB 자동화 개발/유지비 | 인건비 (QA 업무와 통합 가능) |

### 2.2 캡처 가이드 시스템 (신규)

인간 캡처자를 위한 **체크리스트 기반 가이드** 자동 생성:

```
캡처 가이드: Ash N Veil (Idle RPG)
===================================

필수 캡처 항목 (체크리스트):

□ 1. 메인 화면 (HUD 전체 보이게)
□ 2. 캐릭터 스탯 화면 (HP/ATK/DEF 표시)
□ 3. 장비 화면 (전체 슬롯 + 장착 장비)
□ 4. 장비 상세 (아무 장비 1개 탭)
□ 5. 장비 강화 화면 (강화 비용 표시)
□ 6. 스킬 목록 화면
□ 7. 소환/가챠 화면 (비용 + 확률 표시)
□ 8. 상점 화면 (가격표)
□ 9. 맵/스테이지 선택 화면
□ 10. 전투 중 화면 (데미지 숫자 보이게)
□ 11. 전투 결과 화면 (보상 표시)
□ 12. 방치 보상 수령 화면
□ 13. 일일 퀘스트 화면
□ 14. 출석 체크 화면
□ 15. 설정 화면

권장: Lv.1, Lv.10, Lv.30 각각에서 2번 스탯 캡처
(성장 공식 회귀분석용)
```

### 2.3 장르별 캡처 가이드 템플릿

| 장르 | 필수 캡처 수 | 핵심 화면 |

### 3.3 신규 항목

## 5. 미션 재설계: 데이터 커버리지 향상

### 5.1 현재 미션의 문제

현재 10명의 AI 테스터가 **같은 16장의 스크린샷**을 다른 관점으로 보고 있음.
→ 스크린샷에 없는 정보(가챠, 상점, 방치보상)는 10명이 봐도 추출 불가.

### 5.2 개선: 화면 커버리지 우선 미션 설계

**원칙**: "관점의 다양성"보다 "화면의 다양성"이 중요

| 구분 | v2.5 (관점 우선) | v3.0 (화면 우선) |
|------|:----------------:|:----------------:|
| 캡처 소스 | 16장 (중복 화면 포함) | 15~20장 (모든 주요 화면) |
| null 비율 | 53% (17/32) | 목표: < 20% |
| 가챠 데이터 | 미확인 (4/4 null) | 필수 캡처 대상 |
| 방치 보상 | 미확인 (4/4 null) | 필수 캡처 대상 |
| 상점/재화 | 부분 확인 (3/5 null) | 필수 캡처 대상 |

### 5.3 v3.0 배치 구조

```
배치 A: 게임플레이 + UI 매핑
  - 스크린샷: 메인화면, 전투, 맵, 설정
  - 추출: 진행구조 (ANV01~05), 시스템 (ANV30~32)

배치 B: 수치 + 성장
  - 스크린샷: 캐릭터 스탯 (Lv.1, Lv.10, Lv.30+)
  - 추출: 캐릭터 성장 (ANV06~10), 전투 (ANV15~20)

배치 C: 시스템 + 경제
  - 스크린샷: 장비, 강화, 소환, 상점, 방치보상, 퀘스트
  - 추출: 장비 (ANV11~14), 가챠 (ANV21~24), 경제 (ANV25~29)
```

## 8. 결론

C10+ v3.0의 핵심은 **"ADB 자동화를 버리고 인간 캡처로 전환"**하는 것입니다.

이로 인해:
1. **Unity 게임 호환성 문제 해결** (문제 1)
2. **APK 추출 법적 리스크 제거** (문제 2)
3. **분석 비용 53% 절감** (문제 3)
4. **데이터 커버리지 대폭 향상** (null 53% → < 20%)

추가되는 인건비(QA 담당자 10분 캡처)는 자동화 개발/유지비보다 낮으며,
캡처 가이드 시스템으로 표준화하면 누구나 일관된 품질로 캡처 가능합니다.
