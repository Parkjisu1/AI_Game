### 데미지 공식 (코드 완전 추출)

```
CharacterPlayer.cs:650-720

damage = (baseATK + buffATK)             // 1단계: 기본 공격력 + 버프
       × (1 + criDamage)                 // 2단계: 크리티컬 (발동 시)
       × (1 + superCriDamage)            // 3단계: 슈퍼크리티컬 (발동 시)
       × (1 + finalAtkIncrease)          // 4단계: 최종 공격력 증가
       × (1 + weakness)                  // 5단계: 약점 보정
       - flatDamageReduction              // 6단계: 고정 피해 감소
       - percentDamageReduction            // 7단계: 비율 피해 감소
```

### 크리티컬 시스템 (코드 확정)

| 판정 방식 | `Random.Range(0,100000) < percent×100000` | `ExtensionMethods.cs:1396-1399` |

## 17. Smart PlayEngine 아키텍처 (v2.0 신규)

### 5-Level Decision Cascade

```
PlayEngine._tick()
    ↓
[Level 1] GOAP Reasoner
    GoalReasoner → UtilityScorer → GOAPPlanner → Action
    RPG 목표 5종: heal, equip_upgrade, do_quest, grind, summon
    ↓ (action 없으면)
[Level 1.5] PatternExecutor
    PatternDB(3 patterns) → try_match(screen, OCR) → execute_step()
    학습 패턴: quest_main(4), popup_reward(10), upgrade_skill(4)
    4-priority 매칭: OCR텍스트 → NavElement → Intent키워드 → 좌표fallback
    ↓ (매칭 없으면)
[Level 2] ScreenActionResolver
    NavGraph 17노드 → in_screen_actions 라운드로빈
    interaction(tap) + scroll(swipe) 액션 실행, idle 스킵
    오버레이 감지 → close_overlay (nav edge → close button → hub)
    ↓ (액션 소진 시)
[Level 3] ActivityScheduler
    battle(15틱) → lobby(8틱) → menu(3틱) 로테이션
    30틱마다 강제 마을 방문
    ↓ (목표 없으면)
[Level 4] Explore Fallback
    5×8 그리드 탐색 (최후 수단)
```

### Stuck Escape 전략 (6단계)

```
_handle_stuck(screen_type)
    ↓
[Strategy 1] unknown → 게임 재시작 (크래시 복구)
    force-stop → relaunch → 20초 대기
    ↓
[Strategy 2] closeable → 오버레이 닫기
    nav edge close_button → hub edge → fallback tap
    ↓
[Strategy 3] resolver → 최고 성공률 edge 직접 탭
    stage_select→battle @(540,1060), battle→menu_shop @(282,211)
    ↓
[Strategy 4] navigator → hub(lobby/battle) 네비게이션
    ↓
[Strategy 5] fallback → 공통 닫기 좌표 탭
    (540,1870), (1020,80), (60,80)
    ↓
[Strategy 6] random → 랜덤 영역 탭
```
