## Tap Shift Session 04 - 타이밍 분석 보고서

### 핵심 관찰: 5장 모두 거의 동일한 상태

---

### 화면 구성 요약

모든 5장의 스크린샷이 **동일한 화면 상태**를 보여줍니다:
- 레벨 맵 화면 (레벨 14, 16, 17 노드 + 핑크 볼 현재 위치)
- 하단 **PLAY** 버튼
- 우측 상단 **No Ads!** 배너
- 우측 중단 **Claim 선물박스** 버튼
- 화면 중앙 **시스템 GDPR/이용약관 팝업** (OS 레벨 다이얼로그)

---

### 1. 오브젝트 이동 애니메이션

| 항목 | 분석 결과 |
|------|-----------|
| 이동 감지 | **없음** - 01→05 사이 레벨 노드, 볼 위치 모두 고정 |
| "before/during/after_move" 라벨 | 게임 내 이동이 아닌 **캡처 세션 타이밍 레이블**로 추정 |
| 실제 이동 상태 | 이동 전 정지 상태가 모든 프레임에서 유지됨 |

> **추정 이동 시간: 측정 불가 (이동 애니메이션 미포착)**

---

### 2. 이동 가속/감속 패턴

- 이동 중간 프레임이 없어 패턴 추론 불가
- 다른 세션 기준 일반적인 퍼즐 게임의 슬라이드 이동은 **ease-out** (시작 빠름 → 도착 직전 감속) 패턴이 표준
- 추정: `ease-out`, ~0.15~0.25s

---

### 3. 이동 중 변형 (Stretch / 회전)

- 이동 캡처 없음 → 직접 확인 불가
- 볼 오브젝트 형태: 원형 유지, 변형 흔적 없음

---

### 4. 유일하게 감지된 애니메이션: Claim 선물박스 Idle 펄스

5장 간 **미세한 차이가 감지된 유일한 요소**:

| 프레임 | Claim 박스 상태 |
|--------|----------------|
| 01, 02, 03 | 리본/볼륨 기본 크기 |
| 04 | 리본 색상/위치 미세 변화 (약간 확대된 듯) |
| 05 | 01~03 상태로 복귀 |

```
Idle 펄스 패턴 추정:
Scale 1.0 → Scale 1.05~1.08 → Scale 1.0
주기: ~1.0~1.5s (느린 호흡형 pulse)
Easing: ease-in-out (sine curve)
```

---

### 5. 팝업 애니메이션

| 항목 | 내용 |
|------|------|
| 팝업 종류 | OS 시스템 레벨 GDPR/이용약관 다이얼로그 |
| 등장 방식 | **첫 프레임(01)부터 이미 완전히 표시됨** |
| 등장 캡처 여부 | 없음 - 팝업 등장 애니메이션 미포착 |
| 04_popup_appearing vs 05_popup_visible | **두 장 모두 동일한 완전 표시 상태** |
| 예상 등장 방식 | OS 다이얼로그 → 즉시(instant) 또는 0.1s fade-in |

- PLAY 버튼 탭 이후 씬 전환 미포착

### 7. 피드백 애니메이션

- 막힘/오류 상태 없음 → 흔들림 피드백 미관찰

### 세션 한계 요약

```
[제약사항]
- 5장 모두 동일한 정지 상태 캡처
- 이동 애니메이션 중간 프레임 없음
- 팝업 등장 직전 프레임 없음
- OS 레벨 시스템 팝업이 게임 UI 대부분을 가림

[신뢰 가능한 수치]
- Claim 박스 Idle 펄스: 주기 ~1.0~1.5s, scale 변화 ~5~8%
- 시스템 팝업: 완전 표시 (등장 타이밍 불명)
```

### 권장 추가 캡처

이 세션의 데이터 가치를 높이려면:
1. **PLAY 버튼 탭 후** 씬 전환 캡처 (타이틀→게임)
2. **팝업 "계속하기" 탭 직후** 팝업 퇴장 캡처 (0.05s 간격)
3. **레벨 노드 탭 후** 볼 이동 시작 프레임부터 연속 캡처
