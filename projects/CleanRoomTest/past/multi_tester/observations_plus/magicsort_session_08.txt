### 로딩 애니메이션에서 추론 가능한 단서

**확실한 관찰:**
- 하단 점(`.` `..` `...`) 패턴이 순환 → 단순 타이머 기반 dot animation (프레임 기반 또는 `InvokeRepeating`)
- 세션 ID가 화면에 표시됨 → **디버그 빌드** 또는 테스터 전용 빌드

**추론:**
- 로딩이 5장 내내 완료되지 않음 → 초기 리소스 로드가 무거운 구조 (번들 다운로드 가능성)
- ToS 팝업이 로딩 완료 전에 표시됨 → 팝업 트리거는 앱 최초 실행 플래그(PlayerPrefs) 기반, 로딩과 비동기 병렬 처리

### 권고사항

세션 08의 이미지는 알고리즘 분석 목적으로 활용 불가합니다. 다음 두 가지를 확인하세요:

1. **캡처 타이밍 문제** - 세션이 ACCEPT 이후의 게임플레이 장면을 담아야 함
2. **캡처 파일 오류** - 올바른 세션 폴더의 이미지인지 확인 필요 (`session_08` 폴더가 실제로 게임 진입 전 단계의 로그인지 재확인)

게임플레이 장면이 포함된 이미지를 제공해주시면 6개 알고리즘 항목 전부 분석 가능합니다.
