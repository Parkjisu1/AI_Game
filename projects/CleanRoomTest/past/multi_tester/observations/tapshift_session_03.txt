## Tap Shift - 세션 3 분석 (다중 탭 플레이 5+)

---

### 중요 관찰: 3장 스크린샷 동일

**`01_start.png`, `02_mid.png`, `03_after_many.png` 세 장이 픽셀 단위로 완전히 동일합니다.**

- 코인 수: 전부 `85` (변화 없음)
- 레벨 맵 상태: 전부 동일
- UI 레이아웃: 전부 동일

이는 5회 이상 탭 입력에도 불구하고 **화면 상태가 전혀 변하지 않았음**을 의미합니다.

---

### 1. 게임 정체성

| 항목 | 값 |
|------|-----|
| 화면 유형 | 레벨 맵 (월드 맵 / 스테이지 선택 화면) |
| 배경 | 다크 네이비 블루, 미세한 도트 패턴 |
| 아트 스타일 | 캐주얼 모바일, 카툰 스타일 |
| 현재 씬 | 게임플레이 화면 아님 - 레벨 맵 화면 |

---

### 2. UI 구성

**상단 바 (다크 네이비 헤더)**
| 위치 | 요소 | 값 |
|------|------|-----|
| 좌측 | 골드 코인 아이콘 + 수량 | **85** |
| 좌측 코인 하단 | `+` 버튼 (코인 추가 구매) | 녹색 원형 |
| 우측 | 쇼핑 카트 아이콘 | 상점 진입 |
| 우측 끝 | 기어(톱니바퀴) 아이콘 | 설정 |

**우측 플로팅 버튼**
| 요소 | 설명 |
|------|------|
| `ADS / No Ads!` 버튼 | 빨간+파란 원형, 광고 제거 구매 유도 |
| 선물상자 `Claim` 버튼 | 색색의 선물상자 3개 + `?` 마크, 초록색 "Claim" 버튼 |

**하단**
- 대형 초록색 `CONTINUE` 버튼 (전체 너비의 약 60%)

---

### 3. 레벨 맵 구조

수직 경로(세로 선)로 연결된 원형 레벨 노드:

| 레벨 번호 | 색상 | 특이사항 |
|-----------|------|----------|
| 17 | 회색/블루 (상단 일부만 보임) | 미완료 또는 잠금 상태 |
| 16 | 회색/블루 | 미완료 또는 잠금 상태 |
| **15** | **핑크/빨간색** | **해골(skull) 아이콘** 표시 - 특수 난이도 레벨 |
| **14** | **초록색** | **현재 진행 가능 레벨** (활성화 상태) |

**레벨 노드 색상 코드 (추정)**
- 초록 = 현재 플레이 가능
- 핑크/빨간 + 해골 = 보스 또는 고난이도 레벨
- 회색/블루 = 미해금 또는 완료

---

### 4. 재화/점수 시스템

| 재화 | 현재 값 | 위치 |
|------|---------|------|
| 골드 코인 | **85** | 상단 좌측, 별 모양 아이콘이 있는 금색 코인 |
| 기타 재화 | 없음 | 이 화면에서는 코인만 표시 |

---

### 5. 특수 요소

- **해골(Skull) 아이콘**: 레벨 15 노드 상단에 배치 → 특수 레벨 마커 (하드/보스 레벨 가능성)
- **Claim 버튼**: 데일리 보상 또는 선물 수령 기능
- **No Ads 버튼**: IAP(인앱 결제) 광고 제거 상품

---

### 6. 세션 3 핵심 이슈

> **버그 가능성**: "CONTINUE" 버튼 또는 레벨 노드(14번) 탭 시 5회 이상 탭 입력에도 화면 전환이 발생하지 않음
>
> - 세 장 스크린샷 모두 동일 상태 → 탭 입력이 처리되지 않거나, 게임플레이 화면으로 전환이 안 되는 것으로 보임
> - `CONTINUE` 버튼이 반응하지 않거나, 레벨 진입 조건이 충족되지 않은 상태일 수 있음
> - 코인(85) 변화 없음 → 레벨 진입 비용 차감도 없음
